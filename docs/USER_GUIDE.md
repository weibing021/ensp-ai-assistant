## 典型使用流程

### 1. 环境准备

1. 克隆项目并安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
2. 在 eNSP 中创建好虚拟拓扑，打开设备 Telnet/串口映射。  
3. 在项目根目录下配置 `.env`，填入你的 LLM API Key（如果使用云端 LLM）。  
4. 启动 Web 应用：
   ```bash
   streamlit run src/main.py
   ```

### 2. 添加设备并建立连接

1. 打开浏览器进入 Streamlit 页面。  
2. 在左侧“设备管理”中：
   - 输入设备名（如 `R1`）、IP（一般为 `127.0.0.1`）、Telnet 端口（如 `2000`）。  
   - 点击“添加设备”，设备信息会记录进数据库。  
3. 在“已添加设备”列表中展开设备卡片，点击“连接”，成功后状态变为“已连接”。

### 3. 使用自然语言生成配置并执行

1. 切换到 Tab「指令解析与执行」：
   - 在输入框中写下你的配置需求（可以是中文自然语言）。  
   - 点击“开始解析”，等待 AI 解析任务。  
2. 查看“当前解析结果”（JSON 任务列表），确认无误后：
   - 点击“保存至待执行清单”，将解析结果加入任务历史。  
3. 切换到 Tab「任务执行与手动配置」：
   - 在“待执行任务清单”中选中刚刚保存的任务。  
   - 点击“立即执行该任务”，系统会自动为相关设备生成命令并下发。  
   - 在“执行结果反馈”中查看每台设备的命令执行情况。

### 4. 手动命令下发与多设备远程命令

1. 在 Tab「任务执行与手动配置」下方：
   - 选择目标设备。  
   - 在文本框中输入多行 VRP 命令（每行一条）。  
   - 点击“发送配置”，查看执行成功或失败提示。  
2. 使用“多设备远程命令执行”区域：
   - 勾选多个已连接设备。  
   - 输入一组需要批量执行的操作/查询命令。  
   - 一次性并发执行，按设备查看命令输出。

### 5. 网络故障分析与修复建议

1. 切换到 Tab「故障分析排错」：
   - 在“故障现象描述”中尽量详细描述现象（设备名、方向、是否 ping 通、接口状态等）。  
   - 点击“开始 AI 诊断”。  
2. 系统会：
   - 并发采集每台设备的路由表和接口状态。  
   - 调用 AI 分析收集到的信息和你的描述。  
3. 页面上将展示：
   - 诊断分析报告（文字说明）。  
   - 风险等级。  
   - 建议执行的修复命令列表（可复制、也可在手动 CLI 区执行）。

### 6. 拓扑自动发现与可视化

1. 切换到 Tab「拓扑自动发现」：
   - 选择需要参与拓扑发现的设备。  
   - 点击“开始拓扑发现”。  
2. 系统会：
   - 自动检查并尝试启用所选设备的 LLDP 功能。  
   - 并发采集接口、IP、ARP、MAC、LLDP 等信息。  
   - 构建并持久化当前网络拓扑。  
3. 页面上将展示：
   - 当前拓扑图视图（节点为设备，边为链路，附带端口信息）。
